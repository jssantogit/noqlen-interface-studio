# Block Audit Checklist

- Block scope is complete.
- No next block work started.
- Tool Mode is declared and matches actual optional tooling used.
- Raw evidence is available for serious debugging, validation failures, audits, boundary changes and security-sensitive changes.
- Screen contracts are updated.
- Interface context is updated.
- Validation has been run or failures are documented.
- Repository hygiene has been checked.
- Human review is requested for risky, boundary, security, deploy or release changes.
